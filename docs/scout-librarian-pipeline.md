# Scout-Librarian Pipeline

## Overview

Every capture that passes through `ProcessQueue` is routed through an officer pipeline: a **Scout** classifies it, and a **Librarian** files it to the vault. Both are Go interfaces, so custom implementations can replace the defaults.

## Interfaces

Defined in `go/internal/wraith/officer_pipeline.go`:

```go
type ScoutOfficer interface {
    Assess(c *Capture) ScoutAssessment
}

type LibrarianOfficer interface {
    File(vaultDir string, cap *Capture, frontmatter map[string]interface{}, body string) (FilingReceipt, error)
}
```

### ScoutAssessment

```go
type ScoutAssessment struct {
    Class   string  // "keep", "discard", "mission_candidate"
    Reason  string  // Human-readable explanation
    Officer string  // Officer identifier
    Model   string  // Model used for assessment (if any)
    At      string  // RFC3339 timestamp
}
```

### FilingReceipt

```go
type FilingReceipt struct {
    VaultPath string  // Full path to the written file
    Officer   string  // Officer identifier
    Model     string  // Model used (if any)
    Checksum  string  // SHA-256 of the body (first 12 bytes, hex)
    At        string  // RFC3339 timestamp
}
```

## Default Scout Rules

The `defaultScoutOfficer` applies these rules in order:

1. If URL contains `github.com`, or title contains `release` or `cve-` (case-insensitive): classify as **mission_candidate** with reason "signal suggests actionable implementation or security follow-up"
2. If body text, selected text, and title are all empty: classify as **discard** with reason "empty capture payload"
3. Otherwise: classify as **keep** with reason "baseline intake; retain for triage"

The scout model name is loaded from MODUS config (`moduscfg.DefaultAssignment("scout")`), overridable in the config file under `officers.scout.model`.

## Default Librarian Filing

The `defaultLibrarianOfficer` writes markdown files to the vault:

### Directory Structure

The target directory is determined by capture content:

- Tweet captures (`cap.Tweet != nil`): `{vaultDir}/brain/x/`
- Reddit captures (URL contains `reddit.com`): `{vaultDir}/brain/reddit/`
- All other captures: `{vaultDir}/brain/captures/`

### Filename

Format: `YYYY-MM-DD-{slug}.md`

The slug is generated by `slugify()`: lowercase, strip non-word characters, replace whitespace/underscores with hyphens, collapse consecutive hyphens, cap at 80 characters. If the title produces an empty slug, falls back to `capture-{ID}`.

### Frontmatter Fields

The consumer (`consumer.go`) builds frontmatter before passing it to the Librarian:

| Field | Source |
|-------|--------|
| `source` | Capture source (overridden to `x-extension` for tweets) |
| `url` | Capture URL (reconstructed for tweets as `https://x.com/i/status/{tweetID}`) |
| `date` | Current date (YYYY-MM-DD) |
| `triage` | Always `"pending"` at filing time |
| `captured_at` | Original capture timestamp |
| `capture_id` | Capture ID |
| `author` | From capture or tweet handle |
| `site_name` | From capture metadata |
| `tweet_id`, `likes`, `retweets`, `media` | Tweet-specific fields |
| `has_selection` | True if user selected text |
| `scout_class`, `scout_reason`, `scout_officer`, `scout_model` | Added by consumer after scout assessment |

### Checksum

The Librarian computes `SHA-256` of the markdown body and includes the first 12 bytes (24 hex chars) in the `FilingReceipt.Checksum` field.

## Handoff Record

Every officer pipeline execution -- whether the capture is kept, discarded, or fails -- is appended to `wraith-officer-handoffs.jsonl` in the data directory. Each line is a JSON object:

```json
{
  "capture_id": "a1b2c3d4e5f6",
  "fingerprint": "abc123...",
  "source": "context-menu",
  "url": "https://example.com/article",
  "title": "Example Article",
  "scout": {
    "class": "keep",
    "reason": "baseline intake; retain for triage",
    "officer": "scout",
    "model": "rule-based",
    "at": "2026-04-07T12:00:00Z"
  },
  "librarian": {
    "vault_path": "/path/to/brain/captures/2026-04-07-example-article.md",
    "officer": "librarian",
    "model": "rule-based",
    "checksum": "a1b2c3d4e5f6a1b2c3d4e5f6",
    "at": "2026-04-07T12:00:01Z"
  }
}
```

For discarded captures, the `librarian` field is `null`. For failed captures, no handoff record is written (the failure is recorded on the capture's `IngestHistory` instead).

## Implementing Custom Officers

To implement a custom Scout:

```go
type myScout struct{}

func (s myScout) Assess(c *Capture) ScoutAssessment {
    // Your classification logic here
    return ScoutAssessment{
        Class:   "keep",
        Reason:  "custom assessment",
        Officer: "my-scout",
        Model:   "custom-model-v1",
        At:      time.Now().UTC().Format(time.RFC3339),
    }
}
```

Pass custom officers to `ProcessQueueWithOfficers`:

```go
n, err := ProcessQueueWithOfficers(queue, state, vaultDir, limit, OfficerPipeline{
    Scout:     myScout{},
    Librarian: myLibrarian{},
})
```

The test suite (`consumer_test.go`) demonstrates this pattern with `testScoutOfficer` and `testLibrarianOfficer`.
